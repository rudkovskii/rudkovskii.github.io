# Контракт модулей питомца — v2 (assets/pets/*.js)

С v2 у каждого питомца может быть до **4 арт-стилей**: `soft` (базовый),
`cartoon`, `real`, `pixel`. Контракт описывает два вида модулей:

1. **Базовый модуль** `assets/pets/<id>.js` — существующие 5 файлов.
   Остаются **нетронутыми** и автоматически считаются стилем **`soft`**.
2. **Стилевой модуль** `assets/pets/<id>.<style>.js`, где
   `style ∈ cartoon | real | pixel`. Новые файлы, по одному на пару
   питомец×стиль (итого до 15).

## 1. Базовый модуль (стиль soft) — без изменений

```js
window.registerPet({
  id: "cat",                // строго один из списка ниже
  nameEn: "Mila",
  nameRu: "Мила",
  render: function (mood) { // mood — строго одно из 4 значений ниже
    return "<svg ...>...</svg>"; // строка с ОДНИМ анимированным inline-SVG
  }
});
```

## 2. Стилевой модуль (новое в v2)

Файл `assets/pets/<id>.<style>.js` при загрузке вызывает:

```js
window.registerPetStyle({
  petId: "cat",             // id существующего питомца (список ниже)
  style: "pixel",           // строго: "cartoon" | "real" | "pixel"
  render: function (mood) { // те же 4 mood, те же правила SVG
    return "<svg ...>...</svg>";
  }
});
```

Правила:

- `app.js` определяет и `window.registerPet`, и `window.registerPetStyle`
  **до** загрузки любых модулей питомцев (порядок скриптов в `index.html`).
- Стилевой модуль **не** несёт имён (`nameEn`/`nameRu`) — имена живут только
  в базовом модуле и `pets.js`.
- **Fallback обязателен на стороне app.js**: если стилевой модуль не загружен
  или его `render` упал/вернул не-SVG, рендерится стиль `soft` того же
  питомца (а если и его нет — placeholder-blob). Модуль может подключаться в
  `index.html` до того, как файл вообще существует, — сайт обязан работать.
- `render` — чистая функция без побочных эффектов, как в v1.

### Scoping стилевых модулей (обязательно)

SVG разных стилей и питомцев живут в одном DOM одновременно
(сцена + hero + карточки + модалка), поэтому:

- корневой класс: `class="pet-svg pet-<id> pet-<id>-<style> is-<mood>"`,
  например `class="pet-svg pet-cat pet-cat-pixel is-tired"`;
- каждый CSS-селектор в `<style>` внутри SVG начинается с
  `.pet-<id>-<style>` (лучше `.pet-<id>-<style>.is-<mood>`);
- имена `@keyframes` — с префиксом `<id>-<style>-`:
  `cat-pixel-breathe`, `otter-real-float`;
- атрибуты `id=` внутри SVG (градиенты, маски) — с префиксом
  `<id>-<style>-<mood>-`: `id="cat-pixel-radiant-glow"`. Лучше без id вовсе.

Базовые модули (`soft`) сохраняют префиксы v1 (`.pet-<id>`, `cat-breathe`) —
их не переименовываем.

## 4. Mood-значения (одинаковы для ВСЕХ стилей)

`render(mood)` обязан поддерживать ровно эти 4 строки — все без исключения,
заглушки «пока одна поза» запрещены:

| mood | recovery | Что обязано читаться за 1 сек без текста |
|---|---|---|
| `radiant` | 80–100 | открытые глаза с бликом, приподнятая поза, игривое движение, искры |
| `steady`  | 60–79  | спокойное дыхание, мягкая довольная мимика, ровная поза |
| `tired`   | 35–59  | полуприкрытые глаза (веко 40–50%), ссутулен/осел, редкий зевок |
| `drained` | 0–34   | лежит клубком/накрыт пледом, глаза-щёлочки, очень медленное дыхание. Нежность, НЕ страдание: без слёз, туч, красного |

Других mood нет. Режим «дремлет» (F2) app.js реализует сам через
`render('drained')` — отдельная поза сна не нужна ни в одном стиле.

**Смена стиля не меняет mood и питомца** — это делает app.js, модулям об этом
знать не нужно.

## 5. Требования к возвращаемому SVG (все стили)

1. **Одна строка = один `<svg>`**, без обёрток-`div`, без `<script>` внутри.
2. **viewBox фиксированный: `viewBox="0 0 240 240"`.** Атрибуты `width`/`height`
   НЕ ставить — масштабирует CSS страницы. Питомец центрирован, «земля» ~ y=200.
3. **Корневой класс обязателен** (см. Scoping выше).
4. **Все анимации — внутри SVG**: CSS `@keyframes` в `<style>` внутри SVG или
   SMIL. Без внешних зависимостей, без JS, без внешних url(), без картинок.
   Idle-дыхание обязательно во всех 4 mood (питомец никогда не «статуя»).
   Темп: radiant ~2.2s, steady ~3.5s, tired ~5s, drained ~7s.
5. **`prefers-reduced-motion`:** внутри `<style>` SVG обязателен блок
   `@media (prefers-reduced-motion: reduce) { ... animation: none ... }`.
   Статичные позы 4 mood различимы и без движения (поза + веки + аура).
6. **Структура групп (рекомендация):** `<g class="aura">`, `<g class="body">`,
   `<g class="head">`, `<g class="eyes">`, `<g class="eyelids">`,
   `<g class="mouth">`, `<g class="extras">`, `<g class="particles">`.
7. **Бюджет:** ≤ 25 КБ на файл, 25–40 путей на позу (для pixel — до ~250
   rect'ов на позу при сетке, см. ниже), плоские заливки + максимум
   1–2 градиента (pixel — без градиентов вовсе).
8. **Вид животного обязан считываться мгновенно в любом стиле** — силуэт,
   ключевые признаки (уши кошки/кролика, камешек выдры, ветка ленивца,
   «купание» капибары) сохраняются во всех стилях.

## 6. Специфика стилей

### `cartoon` — мультяшный / стикерный
- Жирный **тональный** контур: на 30–40% темнее заливки, толщина 4–6px
  в координатах viewBox. Чистый чёрный `#000` по-прежнему запрещён.
- Ярче и насыщеннее базовой палитры (можно поднять saturation), крупные
  глаза, экспрессивнее мимика и амплитуда анимаций — но темпы mood те же.
- Формы упрощённые, «наклейка»: меньше деталей, чем в soft, больше характера.

### `real` — реалистичная иллюстрация
- Натуральные пропорции и природный окрас животного (реальные цвета вида,
  не пастель) — но это всё ещё тёплая иллюстрация, НЕ фотореализм.
- Мягкая детализация: тени/объём тональными пятнами, допускаются оба
  разрешённых градиента на объём шерсти.
- Обводка тональная тонкая или отсутствует. Анимации сдержаннее по амплитуде.

### `pixel` — пиксель-арт, ретро-тамагочи
- **Приём: сетка из SVG `<rect>`.** Выбери размер пикселя-кванта, кратный
  240: рекомендуем 10 (сетка 24×24) или 12 (сетка 20×20). Каждый «пиксель» —
  `<rect>` с координатами и размерами, кратными кванту. Группируй пиксели
  одного цвета в `<path>`/`<g>` для бюджета, но визуальная сетка строгая.
- На корневом `<svg>` — `shape-rendering="crispEdges"`, никакого сглаживания.
- **Анимации — только `steps()`:** `animation: cat-pixel-breathe 3.5s
  steps(2, end) infinite` и т.п. Кадры — 2–4 позы, переключаемые через
  opacity групп-кадров или transform со сдвигом на целое число квантов.
  Плавных ease/linear трансформаций нет — движение дискретное, как на
  экранчике тамагочи. Темпы mood те же (см. п. 5.4).
- Палитра ограниченная (4–8 цветов на питомца), без градиентов. Дизеринг —
  по желанию, шахматкой из rect'ов.

## 7. Палитра (из визуального кита, раздел 2.2)

Внутри SVG можно использовать CSS-переменные страницы (inline-SVG их наследует)
или напрямую hex из кита:

- Состояния (аура/свечение): radiant `#F5A623` на `#FFF3D9`; steady `#5FAF7E`
  на `#E8F3EA`; tired `#8CA0C9` на `#EDF0F8`; drained `#9C8BC0` на `#F0ECF7`.
  Как переменные: `var(--state-radiant)`, `var(--state-radiant-bg)` и т.д.
- Нейтралы: текстовый графит `#3E3A47` (`--ink`), крем `#FBF6EE` (`--bg`),
  лаванда-подложка `#E9E6FA` (`--calm-soft`), плед — `#E9E6FA`/`#F0ECF7`.
- Цвета самих существ — из раздела 3 визуального кита; в `real` — природный
  окрас вида; в `cartoon` — можно насыщеннее; в `pixel` — редуцированная
  палитра на их основе.
- Запрещено во всех стилях: чистый чёрный `#000`, красные «alert»-цвета.
  Обводка — тональная (см. специфику стиля), не чёрная.

## 8. Тон-фильтр на визуал (обязателен, все стили)

Ни в одном mood и ни в одном стиле: слёз, туч, черепов, пустых мисок,
таймеров, шкал здоровья. `drained` вызывает «хочу укрыть пледом», а не
тревогу или вину.

## 9. Fallback (v2)

- Базовый модуль не загружен → placeholder-blob из `pets.js` (как в v1).
- Стилевой модуль не загружен (или его render упал) → app.js рендерит
  **soft** того же питомца. Пользователь с выбранным стилем никогда не
  видит пустоту.
- Как только `registerPet`/`registerPetStyle` вызван — app.js сам
  перерисовывает сцену, hero и карточки, модулю ничего делать не нужно.

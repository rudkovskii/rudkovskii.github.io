# Контракт модуля питомца (assets/pets/*.js)

Каждый питомец — один файл `assets/pets/<id>.js`, который при загрузке вызывает:

```js
window.registerPet({
  id: "mochi",              // строго один из списка ниже
  nameEn: "Mochi",
  nameRu: "Мочи",
  render: function (mood) { // mood — строго одно из 4 значений ниже
    return "<svg ...>...</svg>"; // строка с ОДНИМ анимированным inline-SVG
  }
});
```

`app.js` определяет `window.registerPet` до загрузки файлов питомцев и сам
перерисовывает сцену при движении слайдеров, смене питомца и смене языка.

## Список id (фиксированный, порядок = pet1…pet5 из ТЗ)

| id | Файл | Имя EN / RU | Существо |
|---|---|---|---|
| `mochi`  | `assets/pets/mochi.js`  | Mochi / Мочи   | круглый кот-булочка (предвыбран) |
| `lumi`   | `assets/pets/lumi.js`   | Lumi / Люми    | розовый аксолотль |
| `fern`   | `assets/pets/fern.js`   | Fern / Ферн    | ленивец с листиком |
| `juno`   | `assets/pets/juno.js`   | Juno / Джуно   | сова-йог |
| `nimbus` | `assets/pets/nimbus.js` | Nimbus / Нимбус| облачная овечка |

## Mood-значения (точный список из ТЗ, раздел 4)

`render(mood)` обязан поддерживать ровно эти 4 строки — все без исключения,
заглушки «пока одна поза» запрещены:

| mood | recovery | Что обязано читаться за 1 сек без текста |
|---|---|---|
| `radiant` | 80–100 | открытые глаза с бликом, приподнятая поза, игривое движение, искры |
| `steady`  | 60–79  | спокойное дыхание, мягкая довольная мимика, ровная поза |
| `tired`   | 35–59  | полуприкрытые глаза (веко 40–50%), ссутулен/осел, редкий зевок |
| `drained` | 0–34   | лежит клубком/накрыт пледом, глаза-щёлочки, очень медленное дыхание. Нежность, НЕ страдание: без слёз, туч, красного |

Других mood нет. Режим «дремлет» (F2, уход с вкладки) app.js реализует сам,
переиспользуя `render('drained')` с мягким оформлением сцены — питомцу
отдельная поза сна не нужна.

## Требования к возвращаемому SVG

1. **Одна строка = один `<svg>`**, без обёрток-`div`, без `<script>` внутри.
2. **viewBox фиксированный: `viewBox="0 0 240 240"`.** Атрибуты `width`/`height`
   НЕ ставить — масштабирует CSS страницы. Питомец центрирован, «земля» ~ y=200.
3. **Корневой класс обязателен:** `class="pet-svg pet-<id> is-<mood>"`,
   например `class="pet-svg pet-mochi is-tired"`.
4. **Все анимации — внутри SVG**: либо CSS `@keyframes` в `<style>` внутри SVG,
   либо SMIL (`<animate>`, `<animateTransform>`). Без внешних зависимостей,
   без JS, без внешних url(), без картинок. Idle-дыхание обязательно во всех
   4 mood (питомец никогда не «статуя»). Темп: radiant ~2.2s, steady ~3.5s,
   tired ~5s, drained ~7s.
5. **Scoping (SVG вставляется в общий DOM, экземпляров может быть несколько —
   сцена, hero, карточки):**
   - каждый CSS-селектор в `<style>` начинается с `.pet-<id>` (лучше
     `.pet-<id>.is-<mood>`), чтобы стили не текли между питомцами;
   - имена `@keyframes` — с префиксом id: `mochi-breathe`, `lumi-gill-sway`;
   - атрибуты `id=` внутри SVG (градиенты, маски) — с префиксом id и mood:
     `id="mochi-radiant-glow"`. Лучше вообще без id, где можно.
6. **`prefers-reduced-motion`:** внутри `<style>` SVG обязателен блок
   `@media (prefers-reduced-motion: reduce) { ... animation: none ... }`.
   Статичные позы 4 mood должны различаться и без движения (поза + веки + аура).
7. **Структура групп (рекомендация из визуального кита):**
   `<g class="aura">`, `<g class="body">`, `<g class="head">`, `<g class="eyes">`,
   `<g class="eyelids">`, `<g class="mouth">`, `<g class="extras">` (жабры /
   листик / перья / облака), `<g class="particles">` (искры, «z»).
8. **`render` — чистая функция:** без побочных эффектов, вызываться может
   многократно и в любом порядке mood.
9. **Бюджет:** ≤ 25 КБ на файл питомца, 25–40 путей на позу, плоские заливки
   + максимум 1–2 градиента.

## Палитра (из визуального кита, раздел 2.2)

Внутри SVG можно использовать CSS-переменные страницы (inline-SVG их наследует)
или напрямую hex из кита:

- Состояния (аура/свечение): radiant `#F5A623` на `#FFF3D9`; steady `#5FAF7E`
  на `#E8F3EA`; tired `#8CA0C9` на `#EDF0F8`; drained `#9C8BC0` на `#F0ECF7`.
  Как переменные: `var(--state-radiant)`, `var(--state-radiant-bg)` и т.д.
- Нейтралы: текстовый графит `#3E3A47` (`--ink`), крем `#FBF6EE` (`--bg`),
  лаванда-подложка `#E9E6FA` (`--calm-soft`), плед — `#E9E6FA`/`#F0ECF7`.
- Цвета самих существ — из раздела 3 визуального кита (у каждого свои).
- Запрещено: чистый чёрный `#000`, красные «alert»-цвета, чёрные обводки
  (обводка — тональная, на 15–20% темнее заливки, или без обводки).

## Тон-фильтр на визуал (обязателен)

Ни в одном mood: слёз, туч, черепов, пустых мисок, таймеров, шкал здоровья.
`drained` вызывает «хочу укрыть пледом», а не тревогу или вину.

## Fallback

Пока файл питомца не загружен/не зарегистрирован, app.js рисует временный
placeholder-blob в цвете питомца из `pets.js` c различимыми mood-позами.
Как только `registerPet` вызван — сцена и карточка перерисовываются настоящим SVG.

# Анкета AI-зрелости для Pinely

Два файла, оба самодостаточные, обоим не нужен интернет после загрузки.

| Файл | Что это | Кому |
|---|---|---|
| `index.html` | Анкета. Проходится за 6–8 минут, в конце человек получает свой уровень и три шага | сотрудникам Pinely |
| `decode.html` | Декодер. Превращает строки из Slack в сводную таблицу и CSV | только тебе |

Живые адреса (после деплоя, см. ниже):

- анкета — https://rudkovskii.github.io/pinely-survey/
- декодер — https://rudkovskii.github.io/pinely-survey/decode.html

---

## Как это работает целиком

1. Ты кидаешь людям ссылку на анкету в Slack.
2. Человек проходит её в браузере. Никуда ничего не отправляется: ни аналитики, ни запросов на сторонние сайты. Все ответы живут только в его вкладке.
3. В конце он жмёт **Copy my result** и присылает тебе одну строку **в личку** (имя в анкете не спрашивается — личка и есть идентификация).
4. Ты собираешь все строки, открываешь `decode.html`, вставляешь их одну под другой и жмёшь **Разобрать**. Получаешь таблицу и кнопку выгрузки в CSV.

Уже выложено — папка `pinely-survey/` лежит в репозитории личного сайта `rudkovskii/rudkovskii.github.io`, у которого Pages включён. Чтобы обновить после правок:

```bash
cd ~/Documents/GitHub/"Одностраничник про меня"
git add pinely-survey

```

3. Подожди 1–2 минуты. GitHub пересобирает сайт сам, никаких кнопок нажимать не нужно.

## Что можно править самому

Всё, что меняется без программиста, лежит в самом верху `index.html` — открой файл в любом текстовом редакторе и ищи строку `const CONFIG`. Ниже неё идут блоки с комментариями на русском. Правило одно: **менять только текст внутри кавычек**, запятые и скобки не трогать.

### Даты воркшопов и хакатона

```js
WORKSHOP_DATES: { w1: 'August 5', w2: 'August 12', w3: 'August 19', hackathon: 'August 26' },
```

Меняешь даты в кавычках — они сами подставятся во все места анкеты.

### Список команд

```js
TEAMS: ['Finance', 'HR', 'Legal Compliance & Tax', 'Market Setup Operations', 'Regional Directors'],
```

Добавить команду — дописать `, 'Название'` перед закрывающей `]`. Вариант **Other** с текстовым полем добавляется сам, его в список писать не надо.

Если добавил команду, под неё стоит завести два промпта — см. `PROMPTS_BY_TEAM` ниже. Если не заведёшь, человеку покажется общий набор `default`, ничего не сломается.

### Разрешить повторное прохождение

```js
ALLOW_RETAKE: false,
```

Поставь `true`, если нужно дать людям пройти анкету заново. При `false` тот, кто уже нажал «Copy my result», при следующем заходе увидит заглушку.

### Вопросы и варианты ответов

Блок `QUESTIONS`. Каждый вопрос выглядит так:

```js
{
  id: 'a1',
  title: 'Текст вопроса',
  hint: 'Пояснение под вопросом',
  options: [
    { t: 'Текст варианта', score: 3 },
    ...
  ]
},
```

- `t` — то, что видит человек. Правь свободно.
- `score` — балл 0–5, по нему считается уровень. **Трогай только осознанно**: варианты намеренно идут вперемешку, чтобы не подсказывать «правильный» ответ, а баллы завязаны на модель уровней.
- `id` менять нельзя — по нему собирается выгрузка.

### Рекомендации и промпты

- `RECOMMENDATIONS` — по три шага на каждый уровень (0–5). Порядок = приоритет, показываются первые три. Если у человека сработал флаг безопасности или «ложной продвинутости», соответствующий шаг встаёт первым автоматически.
- `PROMPTS_BY_TEAM` — по два промпта на команду. Название команды в кавычках слева должно **точно** совпадать со строкой из `TEAMS`.

### Куда присылать результат

```js
DELIVERY: 'Send this line to Dmitrii in a direct message on Slack.',
```

## Как пользоваться декодером

1. Собери из личных сообщений все строки. Каждая начинается с `PINELY2-`.
2. Открой `decode.html` (по ссылке выше или просто двойным кликом по файлу).
3. Вставь строки в поле — **по одной на строку**. Пустые строки и лишние пробелы декодер переживает; если человек прислал строку без префикса `PINELY2-`, тоже разберётся.
4. Жми **Разобрать**.

Получаешь:

- плитки сверху: сколько всего, сколько в группах Light / Medium / Advanced, у скольких сработали флаги, сколько готовы нести задачу на хакатон;
- таблицу по каждому человеку: команда, уровень, четыре оси, флаги, оценка промпта из песочницы, барьеры, интересы;
- частотные списки внизу — что съедает время, где AI уже используют, что мешает, что интересно. **Это и есть материал для программы воркшопов;**
- список «задачи, которые люди готовы отдать AI» — сырьё для выбора процесса под хакатон.

Кнопка **Скачать CSV** отдаёт файл `pinely-ai-survey.csv`. В Google Sheets: `Файл → Импорт → Загрузка`, разделитель определяется автоматически.

Если строка битая (человек скопировал не целиком), декодер напишет её номер и разберёт остальные.

### Что означают флаги

| Флаг | Что значит | Что с этим делать |
|---|---|---|
| `safety gate` | Человек готов загрузить чувствительные данные в публичный AI. Уровень принудительно ограничен двойкой, каким бы ни был остальной результат | Тема данных идёт в первый воркшоп, для этих людей — обязательно |
| `false confidence` | Пользуется активно, а результат не проверяет. Уровень снижен на единицу | В его отчёте верификация уже стоит первым шагом. В программе — блок про проверку |

## Что проверить перед рассылкой

- [ ] Открыть https://rudkovskii.github.io/pinely-survey/ с телефона и пройти до конца — это основной сценарий, люди пойдут по ссылке из Slack.
- [ ] Проверить, что даты воркшопов в `CONFIG.WORKSHOP_DATES` актуальные.
- [ ] Пройти анкету самому и убедиться, что строка результата декодируется в `decode.html`.
- [ ] Скачать CSV и открыть в Google Sheets — проверить, что кириллица и запятые не поехали.
- [ ] Написать текст сопроводительного сообщения в Slack: ссылка, время прохождения, куда присылать результат, и что данные не уходят никуда автоматически.
- [ ] Заранее решить, что отвечать на вопрос «а это анонимно?» — анкета не спрашивает имени, но присылают её в личку, так что честный ответ: «я вижу, кто прислал, сводка будет по командам».

## Технические заметки

- Один файл на страницу, весь CSS и JS внутри. Ни сборки, ни зависимостей, ни npm.
- Ноль внешних запросов: ни CDN, ни шрифтов, ни аналитики. Проверяется в DevTools → Network: после загрузки страницы список пустой. Это требование ИБ Pinely.
- Прогресс сохраняется в `sessionStorage` — случайное обновление страницы не сбрасывает ответы.
- Флаг завершения — единственная запись в `localStorage`, без каких-либо данных ответов.
- Строка результата — base64 от JSON с префиксом `PINELY2-`. Если поменяешь префикс в `CONFIG.PAYLOAD_PREFIX`, поменяй его же в `decode.html` (константа `PREFIX` в начале скрипта).
- Локальный предпросмотр: конфиг `pinely-survey` в `Планы/.claude/launch.json`, порт 8902.
